硬體需求

AMD AI MAX+ 395 執行 LTX 2.3 建議配置
項目 規格 說明
CPU AMD Ryzen AI MAX+ 395 16 Zen 5 核心
GPU 40 CU RDNA 3.5（Radeon 8060S） 搭配 ROCm 相容驅動
記憶體 128GB unified ⚠️ BIOS 設為 64GB/64GB
作業系統 Windows 10/11 推薦 Windows 11
儲存空間 50GB+ 模型 + 依賴 + 輸出
⚠️ RAM 配置警告
你必須在 BIOS 設為 64GB/64GB（CPU/GPU 均分）。 預設 96GB/32GB 常導致 OOM 錯誤。

如何設定（4 步驟）

  1. 安裝 AMD Adrenaline 驅動

    下載最新版驅動並安裝，完成後重啟。

    下載連結

  2. 安裝 Git（如已安裝可跳過）

    驗證：git --version。

    下載連結

  3. 安裝 ComfyUI Desktop

    你可自動安裝 Python、ROCm、PyTorch，首次啟動約 10-20 分鐘。

    下載連結

  4. 調整記憶體分配

    ⚠️ 重啟進入 BIOS → Memory Configuration → 設為 64GB/64GB。

快速開始

  1. 載入範本：File → Load → Templates → LTX Video。
  2. 輸入 Prompt：一隻可愛的橘貓在陽光下打哈欠。
  3. 設定參數：Duration 5 秒，Resolution 1920x1080。
  4. 開始生成：點 Queue Prompt，等待 3-5 分鐘。
  5. 查看結果：完成後在 ComfyUI/output/ 查看。

常見問題

首次啟動很慢，正常嗎？

正常。首次會初始化環境與模型，後續啟動會明顯更快。

出現 "CUDA out of memory" 錯誤？

先確認 BIOS 是否為 64GB/64GB，再把秒數與解析度降一級重試。

生成速度比預期慢？

先用 5 秒、720p 驗證流程。穩定後再提高到 1080p 或更長片段。

可以生成超過 5 秒的影片嗎？

可以。建議逐步拉長秒數，避免一次拉太高導致 OOM。

我的 AMD GPU 不是 AI MAX+ 395，能用嗎？

可以先嘗試，但你需確認 ROCm 相容性與驅動版本。
